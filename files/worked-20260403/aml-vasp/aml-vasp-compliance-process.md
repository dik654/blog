# 가상자산사업자(VASP) AML/CFT 컴플라이언스 프로세스

> 특금법 기반 가상자산사업자의 자금세탁방지(AML) 및 공중협박자금조달 금지(CFT) 의무이행 프로세스 정리.
> 회사 고유 정보 제거. 일반적인 VASP 운영 기준.

---

## 1. AML/CFT 의무이행 기준

### 1.1 고객확인제도 (CDD — Customer Due Diligence)

**고객확인 구성:**
- 신원확인: 실명, 주소, 생년월일, 연락처
- 신원검증: 신분증 기반 본인확인 (eKYC 가능)
- 실제소유자 확인: 법인 고객의 경우 25% 이상 지분 보유자 또는 실질 지배자

**이행 시기:**
- 계정 개설 시
- 일정 금액 이상 거래 시
- 자금세탁 의심 시
- 기존 정보의 정확성에 의심이 있을 때

**CDD 미완료 시 제한:**
- 계정 개설 불가
- 거래(입출금, 매매) 불가
- 시스템상 '제한 상태' 자동 분류

**강화된 고객확인 (EDD):**
- 고위험 고객: 정치적 주요 인물(PEP), 고위험 국가 거주자
- 추가 소명 자료 요청
- 자금 출처 확인
- 거래 목적 확인

### 1.2 고객 자산 분리 관리
- 고객별 거래내역 식별 가능한 방식으로 분리 (입출금, 보유, 거래 상대방)
- 고객 예치금과 회사 고유재산 별도 계좌 관리
- 회계상 분리 기록, 외부감사 시 증빙 가능

### 1.3 거래 제한
- **미신고 VASP**: FIU 신고 미완료 사업자와 영업 거래 금지 → FIU DB 주기적 확인
- **특수관계인 거래**: 회사/특수관계인이 발행한 가상자산 중개 금지
- **임직원 사적 거래**: 회사 플랫폼 통한 사적 거래 제한
- **자기거래**: 중개를 가장한 자기거래 금지

## 2. 위험기반 접근법 (RBA — Risk-Based Approach)

### 2.1 배경
FATF 2012년 新권고사항 — 위험기반 접근법 전면 도입:
- 저위험 분야 → 간소화 조치
- 고위험 분야 → 자원 집중
- 유연한 제도 운영 기반 제공

### 2.2 위험평가 4단계 프로세스

**1단계: 위험 식별**
| 위험 유형 | 평가 항목 |
|---|---|
| 고객 위험 | 고객 유형, 국적, 거주지, PEP 여부 |
| 상품/서비스 위험 | 익명성 수준, 거래 복잡도, 신규 상품 |
| 지역 위험 | FATF 고위험국, 제재국, 부패 지수 |
| 거래채널 위험 | 비대면, 제3자 통한 거래 |

**2단계: 위험 분석**
- 식별된 위험의 발생 가능성과 영향도 평가
- 위험 등급 부여 (고/중/저)

**3단계: 위험 평가**
- 고유 위험 대비 현재 통제 수준 평가
- 잔여 위험 산출

**4단계: 지속적 위험평가**
- 연 1회 이상 전사 위험평가 실시
- 환경 변화 시 수시 재평가

### 2.3 RBA 위험관리 조직 (3선 방어)
| 방어선 | 역할 |
|---|---|
| 1선 (현업) | 고객 접점, 일상적 위험 관리 |
| 2선 (준법감시/위험관리) | 정책 수립, 모니터링, 위반 조사 |
| 3선 (내부감사) | 독립적 감사, 개선안 도출 |

### 2.4 고객 위험평가 모델
- 정성 평가: 전문가 판단 기반
- 정량 평가: 수치화된 점수 기반
- 통제 평가: 정성 × 정량 조합
- 잔여위험 = 고유위험 - 통제효과

## 3. 이상거래 탐지 (FDS)

### 3.1 감시 대상
- 단기간 비정상적 빈도/규모 입출금
- 리워드 시스템 허점 이용 의심 패턴
- 설명되지 않는 특정 자산 비정상 운용
- 특정 계정 그룹의 특이 패턴
- 외부 불공정거래 연관 가능성

### 3.2 자동 탐지 유형
- 고빈도 반복 거래
- 신규 가입자의 고액 전송
- 믹싱 서비스 / 프라이버시 코인 전송
- 첫 입금 후 즉시 전액 출금
- 출금 주소 다수 분산

### 3.3 감지 시 조치 절차
1. 거래 자동 보류 + AML 팀 알림
2. 거래 상세 분석 → 준법감시인 보고
3. 해당 계정 입출금 정지 + 본인확인 재요청
4. 불공정거래 의심 시 → 금융위/금감원 통보
5. 혐의 충분 시 → 수사기관 신고
6. 모든 과정 상세 기록 보관

---

## 4. 의심거래 보고 (STR/SAR)

### 4.1 보고 대상
- 불법재산으로 의심되는 거래
- 자금세탁/테러자금조달 의심 거래
- 범죄수익 신고 대상 거래

### 4.2 보고 절차
1. 임직원이 의심 거래 발견 → 보고책임자에게 즉시 보고
2. 보고책임자가 검토 → SAR 작성
3. FIU(금융정보분석원)에 제출
4. 관련 정보 외부 유출 엄금

### 4.3 SAR 포함 내용
- 거래 시점 및 방법
- 관련 계정, 주소, 출금 대상 정보
- 조치 내역 및 향후 대응계획

---

## 5. 내부통제 체계

### 5.1 핫월렛/콜드월렛 분리 운영
- 고객 자산 대부분 콜드월렛 보관 (80% 이상)
- 핫월렛 일일 출금 한도 제한
- 콜드→핫 전송: 정해진 절차 + 책임자 승인

### 5.2 다중 서명 (Multi-sig) 체계
- 중요 자산 이동 시 2인 이상 승인 필수
- 콜드월렛 접근: 최소 2인 인가 담당자 공동 수행

### 5.3 시스템 접근 통제
- 관리 콘솔/지갑: VPN + MFA 필수
- 최소 권한 원칙 (Least Privilege)
- 로그 전수 기록 및 감사

### 5.4 개발/배포 통제
- Git 기반 버전관리
- 테스트넷 시뮬레이션 후 운영 반영
- 스마트 계약 배포 시 외부 보안 감사

### 5.5 내부 정보 보호
- 미공개 정보(상장 등) 사적 사용 금지
- 내부자 거래 / 프론트러닝 금지 + 감시 체계
- 미공개중요정보 보안등급 '기밀' 분류
- 접근 최소 인원 제한 + 생성~폐기 전 과정 기록

## 6. 가상자산 보관 지침

### 6.1 이용자 자산 관리
- 이용자명부 작성: 주소, 성명, 가상자산 종류/수량, 지갑 주소
- 이용자명부 암호화 저장, 접근 권한 최소화
- 이용자 자산과 회사 자산 별도 지갑에 분리

### 6.2 콜드월렛 보관 비율
- 이용자 가상자산 경제적 가치의 **80% 이상** 콜드월렛 보관
- 매월 초 5영업일 이내 비율 재산정
- 콜드월렛 개인키 → 물리적 안전 장소 보관

### 6.3 외부 수탁 조건
- 사고 예방/위험관리 체계 보유
- 이해상충 방지체계 보유
- 재위탁 금지
- 업무지침 자체 홈페이지 공시

---

## 7. 불공정거래 방지

### 7.1 금지 행위
- 미공개중요정보 이용 거래
- 시세조종
- 부정거래 (허위 정보 제공, 기망적 운용)
- 자기발행 가상자산 거래 (원칙적 제한)

### 7.2 임직원 거래 제한
- 미공개정보 취득 가상자산: 정보 공개 후 일정 시간 경과 전까지 거래 불가
- 위반 시 인사 조치, 징계, 해고 또는 수사기관 신고

---

## 8. 금융사고 대응

### 8.1 내부 사고 (내부자 부정행위, 권한 미흡 등)
1. 관련 지갑 출금 즉시 정지 + 연관 계정 동결
2. 1시간 이내 CCO/CISO/AML 책임자 보고
3. 사고관리시스템 등록
4. SAR 작성 → FIU 제출
5. 내부 감사 → 로그/승인이력/접근기록 조사
6. 외부 블록체인 분석기관 협력 (자산 추적)
7. 재발방지 대책 수립

### 8.2 외부 사고 (피싱, 해킹, 차명계정 등)
1. FDS 이상거래 자동 보류
2. 사고 계정 입출금 정지 + 본인확인
3. SAR → FIU 제출
4. 경찰 사이버수사대, 외부 거래소, 블록체인 분석업체 공조
5. 피해 고객 보호 (보상/환급 검토)

### 8.3 재발방지 제도화
- 내부자 행동 모니터링 (이상 로그인, 업무 외 시간 지갑 접근)
- 핫월렛 이체 한도 자동 조정 (급증 시 자동 보류)
- AI 기반 이상거래 탐지 고도화
- Mixer/Privacy 코인 블랙리스트 정기 갱신
- eKYC + AI 문서 진위검증 도입

## 9. 기록 보관 및 교육

### 9.1 기록 보관
- 고객 확인 자료, 거래 내역, 보고 내역: **5년 이상** 보관
- 모든 감시/분석/조치/보고 과정 상세 기록

### 9.2 정기 교육
- 임직원 AML/CFT 교육: 연 1회 이상
- 출금 승인 권한자: 반기 1회 이상 보안교육
- 교육 내용: 특금법 개요, CDD/STR 절차, 고위험 고객/국가 식별 기준

### 9.3 정기 점검
- 보고책임자: 분기별 의무이행 점검
- 연 1회 외부 감사/자문기관 평가
- 이상거래 탐지 알고리즘: 연 1회 점검 및 수정

## 10. 책임자 체계

| 역할 | 책임 |
|---|---|
| 내부통제책임자 (CCO) | 정책 수립, 위반 조사 및 보고 |
| 정보보안책임자 (CISO) | 시스템 접근통제, 보안 로그 감사 |
| AML 담당자 | 의심거래 보고, 고객정보 검토 |
| 준법감시인 | 내부통제 기준 준수 점검, 위반 조사 |
| 독립적 감사 | 분기별 통제 체계 감사 및 리포트 |
