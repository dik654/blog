# ISMS-P 인증 프로세스 전체 정리

> 회사 고유 정보 제거. 가상자산사업자(VASP) 기준 ISMS-P 인증 시 필요한 절차와 산출물 정리.

---

## 1. 관리체계 수립 및 운영 (1.x)

### 1.1 정보보호정책 수립
- 정보자산 보안등급 산정: 기밀성(1-3) × 무결성(1-3) × 가용성(1-3) 평균값
  - 2.0 초과 → "비밀" / 1.0~2.0 → "대외비" / 1.0 이하 → "일반"
- 위험평가: GAP 분석 기반, 발생 가능성 H/M/L 부여
- 위험관리 방법은 "위험분석 계획서"에 따라 진행

### 1.2 정보자산 분류 기준
| 자산구분 | 설명 |
|---|---|
| Linux/Win 서버 | 서비스 제공 및 운영/관리용 시스템 |
| 스토리지 | iSCSI, NAS, SAN 등 |
| DBMS | 사용자 정보, 서비스 데이터 저장소 |
| Data 월렛 | 지갑주소, 패스프레이즈 등 가상화폐 거래 정보 |
| WEB/APP | 웹서버 미들웨어, 개발 프로그램 |
| 네트워크장비 | 라우터, 스위치, 무선AP |
| 정보보호시스템 | IPS, 방화벽, VPN 등 |
| PC / 지갑전용PC | 업무용 단말기 (지갑전용은 별도 분류) |
| 부대설비 | CCTV, UPS, 출입통제 등 |

### 1.3 조직 역할
| 역할 | 책임 |
|---|---|
| 최고경영자 | 연 1회 위험평가, 정보보호정책 제·개정 |
| 정보보호 최고책임자(CISO) | 관리체계 수립/운영, 취약점 분석, 침해사고 대응 |
| 개인정보 보호책임자(CPO) | 개인정보 처리 계획, 실태 조사, 불만 처리 |
| 정보보호 실무조직 | 보호대책 구현, 교육, 점검 수행 |

### 1.4 위험평가 절차
1. 정보자산 식별 → 중요도 산정
2. 위협정보 수집
3. 위험평가 실시 (연 1회 이상)
4. DoA(수용 가능한 위험수준) 경영진 승인
5. 보호대책 선정 → 이행계획 수립

## 2. 보호대책 요구사항 (2.x) — 항목별 프로세스

### 2.1 정책·조직·자산 관리
- 정보자산 목록 최신화 (추가/파기 시 즉시 반영)
- DB 테이블 목록 등 내부 저장 정보 식별
- 개인정보 수집·보유 항목, 보유량, 처리 목적/방법, 보유기간 현황 정리
- 업무 절차·흐름 문서화 (GAP 분석, 네트워크 구성도 등)
- **산출물**: 정보자산관리대장, 정보보호정책 문서, 네트워크 구성도

### 2.2 인적 보안
- 정보보호서약서: 임직원 및 외주/협력사 대상
- 연간 교육 계획 수립 및 실시 (연 1회 이상)
- 신규 입사자 필수 교육 이수 후 시스템 접근 권한 부여
- **산출물**: 정보보호서약서, 교육계획서, 교육실시 기록

### 2.3 외부자 보안
- 외부 위탁 시 정보보호 협약서 체결
- 위탁업무 범위, 보안 요구사항, 사고 시 책임 명시
- 주기적 위탁사 보안 점검
- **산출물**: 위탁업무 정보보호 협약서, 점검 기록

### 2.4 물리 보안
- 월렛룸(콜드월렛 보관소) 출입통제: 인가자만 접근
- 작업 시 계획서 → 승인 → 실행 → 완료서 제출 프로세스
- CCTV 녹화, 출입 로그 보관
- **산출물**: 출입통제 대장, 작업계획서/완료서, CCTV 보관 정책

### 2.5 인증 및 권한관리
**비밀번호 정책:**
- 생성 규칙: 영문+숫자+특수문자 조합, 최소 8자
- 변경 주기: 90일 (관리자/사용자 동일)
- 장기 미접속자 처리: 마지막 접속일 기준 6개월 이상 → 계정 잠금 → 복구 페이지 이동

**관리자 페이지:**
- 2차 인증(OTP/MFA) 적용 필수
- 중복 로그인 제한
- 비밀번호 초기화 권한은 최종관리자에게만

**계정 관리:**
- 신규 계정 신청서 기반 발급
- 공용계정 사용대장 관리
- 퇴직 시 즉시 계정 비활성화
- **산출물**: 계정 신청서, 공용계정 사용대장, 2차인증 적용 증적

### 2.6 접근통제
**DB 접근제어 체계:**
- 접근제어 소프트웨어(PETRA 등) 도입
- 망분리 PC에서만 UI 접근 허용
- 세션 접근 제어 + 쿼리 접근 제어

**DB 계정 분리 원칙:**
| 계정 유형 | 권한 | 용도 |
|---|---|---|
| 서비스 계정 (web_user 등) | readWrite, 특정 DB만 | 웹 서비스 전용 |
| 관리자 계정 (admin) | readWrite, 작업 기간 한정 | 작업신청서 기반 임시 권한 |
| 백업 계정 (backup) | select only | SQL 백업 전용 |
| 접근제어 로그 계정 | select only | 접근제어 소프트웨어 로그용 |
| 슈퍼관리자 | 전체 | 비상용 (팀장급 이상) |

**IP 기반 접근 제한:**
- 개발자 PC → 개발 서버만 접근
- 서비스 서버 → 서비스 DB만 접근
- 관리자 IP → 접근제어 UI 허용

**월간 로그 검토:**
- 매월 DB 접근 로그 검토서 작성
- 비인가 접근 시도 확인
- **산출물**: 접근제어 정책 문서, 계정 분리 증적, 월간 로그 검토서

### 2.7 암호화 적용
**비밀번호 암호화:**
- 단방향 해시 알고리즘 적용 (bcrypt, scrypt 등)
- 기존 취약 알고리즘(MD5, SHA-1) → 강화 알고리즘으로 마이그레이션

**키 관리:**
- AWS Secrets Manager 등 외부 키관리 서비스 활용
- 소스코드에 키/시크릿 하드코딩 금지
- 환경변수 또는 시크릿 매니저를 통한 주입
- **산출물**: 암호화 적용 증적, 키관리 정책

### 2.8 정보시스템 개발 보안
**웹 보안 조치:**
- nginx 서버 토큰 노출 방지: `server_tokens off;`
- 에러 페이지 커스텀 처리 (스택 트레이스 노출 금지)
- UUID 적용: 순차 ID → UUID로 변경하여 열거 공격 방지

**세션/인증:**
- 세션 고정 방지
- CSRF 토큰 적용
- XSS 필터링

**개발/배포 프로세스:**
- Git 기반 버전관리 필수
- 테스트 환경에서 검증 후 운영 반영
- 코드 리뷰 + 보안 검수
- **산출물**: 보안 조치 증적, 코드 리뷰 기록

### 2.9 시스템 및 서비스 운영관리
- 서비스 변경 시 변경관리 프로세스 준수
- 장애 발생 시 즉시 보고 → 원인 분석 → 재발 방지
- 운영/개발 환경 분리

### 2.10 시스템 및 서비스 보안관리
- 정보보호시스템(방화벽, IPS) 정책 주기적 검토
- 패치 관리: 보안 업데이트 정기 적용
- 업무용 단말 보안 관리

### 2.11 사고 예방 및 대응
**침해사고 대응 절차:**
1. 탐지 → 초동 대응 (시스템 격리)
2. 분석 → 원인 규명
3. 복구 → 정상 서비스 복원
4. 재발방지 → 보안 강화

**정보유출 조치:**
- 유출 경로 차단
- 영향 범위 파악
- 유관기관 신고 (KISA, 금감원 등)
- 이용자 통지
- **산출물**: 침해사고 대응 절차서, 사고 보고서

### 2.12 재해복구
**백업 정책:**
| 대상 | 주기 | 보관기간 | 방법 | 저장소 |
|---|---|---|---|---|
| DB (MariaDB 등) | 매일 | 30일 이상 | 자동 mysqldump | 클라우드 스토리지(S3) |
| 서버 설정 (/etc, /opt) | 분기별 | 1년 | 수동 전체백업 | 외장 HDD (금고 보관) |
| 애플리케이션 로그 | 매일 | 6개월 | 자동 증분백업 | 클라우드(S3/Glacier) |

**백업 절차:**
1. NTP 시간 동기화 설정
2. 최초 전체 백업 1회
3. 이후 자동 크론 작업으로 증분 백업
4. 개인정보 포함 데이터는 AES-256 암호화
5. 중요 데이터 소산백업 (지리적 분리)
6. 백업 실패 시 Slack Webhook 등 즉시 알림

**복구 절차:**
- DB 복구: S3에서 덤프 파일 → 동일 버전 DB 설치 → 복원 → 정상 확인
- 시스템 복구: Clonezilla 등 이미지 기반 복구 → 부팅 확인 → 설정 검토

**보존 기간:**
- 일반 데이터: 최소 6개월
- 법적 요구사항 데이터: 최대 3~5년
- CCTV: 별도 정책
- **산출물**: 백업정책 문서, 복구 테스트 기록

## 4. ISMS 인증심사 보완조치 프로세스

### 보완조치 흐름
1. 심사 → 결함 도출 (항목번호 부여, 예: 2.5.6, 2.7.1)
2. 결함 정리표 작성 (항목별 결함 내용, 심각도)
3. 보완조치 내역서 작성 (결함별 조치 내용, 증적)
4. 보완조치 완료 시 → 완료 공문 제출
5. 기간 연장 필요 시 → 연장 공문 + 요약서 제출
6. 보완조치 결과 검토 → 이행 확인

### 필요 양식
- 보완조치내역서 (항목번호별)
- 보완조치 완료 공문
- 보완조치 연장 공문
- 보완조치 요약서 (단일심사용)
- 보완조치완료확인서
